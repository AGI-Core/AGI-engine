## 通俗类比库（术语翻译器）

用于将专业术语翻译成零基础用户也能秒懂的大白话。

### Go/tRPC 生态术语（重点）

| 术语 | 通俗翻译 | 类比 |
|------|----------|------|
| tRPC | 腾讯内部的微服务通信框架 | 就像快递公司，负责把请求从一个服务送到另一个服务 |
| proto 文件 | 接口契约/说明书 | 就像菜单，写清楚"有什么服务、需要什么参数、返回什么结果" |
| pb.go | proto 自动生成的 Go 代码 | 就像菜单翻译成厨房能看懂的工单 |
| service（tRPC） | 服务的入口层 | 就像餐厅前台，接待客人、分配任务 |
| handler | 请求处理函数 | 就像服务员，接到点单后去找对应的厨师 |
| filter/拦截器 | 请求的安检通道 | 每个请求进来前都要过一道检查（鉴权、限流、日志等） |
| naming/服务发现 | 通讯录 | 知道每个服务的"电话号码"（IP地址），才能打过去 |
| codec | 翻译官 | 把数据从一种格式翻译成另一种格式（序列化/反序列化） |
| goroutine | 轻量级小工人 | Go 语言的并发单位，一个程序可以同时派出几万个小工人干活 |
| channel | 传话筒 | goroutine 之间传递消息的管道 |
| context | 请求的身份证 | 携带请求的超时时间、取消信号、链路追踪信息等 |
| defer | 离开前记得关门 | 函数结束前一定会执行的清理操作 |
| interface | 能力证书 | 只要你有这个证书（实现了这些方法），就能上岗 |
| struct | 数据容器 | 把相关的数据打包在一起，像一个表格的一行 |
| go.mod | 购物清单 | 记录项目依赖了哪些外部包及其版本 |
| trpc_go.yaml | 服务的配置文件 | 告诉服务"你叫什么名字、监听哪个端口、连哪些下游" |
| client proxy | 远程遥控器 | 用来调用其他服务的客户端，像遥控器一样操控远端服务 |
| redis proxy | Redis 遥控器 | 用来操作 Redis 缓存的客户端 |
| UDF | 用户自定义函数/条件 | 像插件一样，可以动态加载的业务逻辑 |
| 大仓/Monorepo | 超级仓库 | 把多个服务的代码放在同一个 Git 仓库里管理 |

### 前端术语

| 术语 | 通俗翻译 | 类比 |
|------|----------|------|
| 组件 | 积木块 | 一个页面由很多积木拼成，每个积木可以单独用 |
| 路由 | 导航地图 | 告诉程序"这个地址去哪个页面" |
| 状态管理 | 全局记事本 | 所有页面都能读写的共享数据 |
| Props | 父母给孩子的礼物 | 父组件传给子组件的数据，只能看不能改 |
| State | 自己的日记本 | 组件自己的数据，可以随时改 |
| 生命周期 | 人的一生 | 组件从出生（挂载）到死亡（卸载）的过程 |
| 虚拟 DOM | 草稿纸 | 先在草稿上画好，再一次性抄到正式纸上 |
| Hooks | 钩子 | 在特定时机"钩"住，执行你想做的事 |
| SSR | 厨房出餐 | 服务器把页面做好端上来，不是让顾客自己拼 |
| CSR | 自助餐 | 浏览器自己组装页面 |

### 后端术语

| 术语 | 通俗翻译 | 类比 |
|------|----------|------|
| API | 服务员 | 前端点菜、后端做菜、API 传菜 |
| 中间件 | 安检通道 | 请求进来前先过一道检查 |
| 数据库 | 仓库 | 存放所有数据的地方 |
| ORM | 翻译官 | 把代码语言翻译成数据库语言 |
| 缓存 | 便利贴 | 把常用信息贴在手边，不用每次去仓库找 |
| 队列/MQ | 排队叫号 | 任务太多先排队，一个一个处理 |
| 微服务 | 专科医院 | 每个服务只干一件事，各司其职 |
| 网关 | 大门保安 | 所有请求先过网关，统一检查和分发 |
| 鉴权 | 查身份证 | 确认"你是谁"以及"你能干什么" |
| 事务 | 要么全做要么全不做 | 银行转账：扣钱和加钱必须同时成功 |
| TCC | 三步走事务 | Try(预留)→Confirm(确认)→Cancel(回滚)，分布式场景下保证一致性 |
| 分布式锁 | 公共厕所的门锁 | 同一时间只能一个人用，防止并发冲突 |
| 幂等 | 按电梯按钮 | 按一次和按十次效果一样，不会重复执行 |
| 降级/兜底 | 备用方案 | 主方案挂了，自动切换到备用方案 |
| 限流 | 景区限流 | 人太多就不让进了，保护系统不被压垮 |
| 熔断 | 保险丝 | 下游服务挂了就断开连接，不要一直重试拖垮自己 |

### 架构模式术语

| 术语 | 通俗翻译 | 类比 |
|------|----------|------|
| 分层架构 | 流水线 | 每层只干自己的事，按顺序传递 |
| 依赖注入 | 外卖模式 | 需要什么别自己做，让别人送过来 |
| 接口/Interface | 插座标准 | 只规定"长什么样"，不管里面怎么实现 |
| 工厂模式 | 自动贩卖机 | 告诉它要什么，它自动给你造一个 |
| 单例模式 | 全校只有一个校长 | 整个程序只有一个实例 |
| 观察者模式 | 订阅公众号 | 有新消息就自动通知所有订阅者 |
| 策略模式 | 导航选路线 | 同一个目的地，可以选不同的路线（策略） |
| 注册模式 | 挂号 | 先注册自己的能力，需要时按名字找到你 |

### 通用编程术语

| 术语 | 通俗翻译 | 类比 |
|------|----------|------|
| 变量 | 盒子 | 用来装数据的容器，可以换内容 |
| 函数 | 机器 | 放进原料，出来产品 |
| 回调函数 | 留个电话 | 事情办完了打给你 |
| Promise | 承诺书 | 承诺未来给你结果（成功或失败） |
| async/await | 排队等号 | 告诉程序"这里要等一下" |
| 继承 | 家族遗传 | 子类自动拥有父类的能力 |
| 多态 | 一招多用 | 同一个方法，不同对象有不同表现 |
| 递归 | 俄罗斯套娃 | 自己调用自己，一层套一层 |
| 并发 | 一心多用 | 同时处理多件事（不一定真的同时，可能是快速切换） |
| 并行 | 多人同时干活 | 真的同时处理多件事（多核CPU） |
